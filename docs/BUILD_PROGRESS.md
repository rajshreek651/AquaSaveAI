# AquaSave AI Build Progress

## Step 1 - Define the problem - Complete

Created the problem statement, target users, measurable success criteria, and transparent simulation assumptions. This gives the project a defensible real-world purpose before code is written.

## Step 2 - Create realistic simulation data - Complete

Generated 336 hourly household-use records across 14 days. The dataset includes normal activity, a simulated overnight leak, and an unusually high garden-use event so the detection logic has meaningful cases to identify.

## Step 3 - Analyse data and detect wastage - Complete

Created a repeatable Python analysis script. It uses clear rules to detect sustained overnight flow and statistically unusual hourly use, then estimates water, cost, energy, and CO2 impact.

## Step 4 - Build the dashboard and conservation simulator - Complete

Built a dependency-free browser dashboard. It visualises daily use, shows every alert with its reason and recommended action, and lets a user simulate monthly savings from leak repair and shorter showers.
